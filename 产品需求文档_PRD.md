# 基于禅道的 AI 项目管理助手 PRD

版本：v1.0  
日期：2026-07-09  
产品名称：禅道 AI 项目管理助手  
文档类型：产品需求文档 PRD  
MVP 范围：Bug AI 分析、客户反馈 AI 分类、文档问答、项目风险周报、禅道扩展入口与人工确认机制

## 1. 产品概述

### 1.1 产品定位

禅道 AI 项目管理助手是一套基于禅道开源版的 AI 增强系统，通过禅道扩展页面接入独立 AI 服务，帮助研发团队分析 Bug、客户反馈、项目文档和项目风险。

产品不替代禅道，而是增强禅道：

- 禅道负责项目、需求、任务、Bug、测试、文档、反馈等基础管理。
- AI 助手负责分析、总结、归类、检索、生成建议。
- 用户确认后，AI 建议才能写回禅道。

### 1.2 产品目标

MVP 目标：

- 降低 Bug 分析和重复排查成本。
- 提升客户反馈归类和转需求/Bug 的效率。
- 让项目文档可问答、可总结、可检查。
- 帮助项目经理提前发现进度和质量风险。
- 验证“禅道扩展 + 独立 AI 服务”的产品可行性。

### 1.3 非目标

MVP 不做：

- 不替代禅道原有项目管理流程。
- 不自动修改 Bug 状态、负责人、优先级。
- 不自动关闭问题。
- 不做完整 BI 报表平台。
- 不做移动端。
- 不做复杂多租户商业后台。

## 2. 用户角色

| 角色 | 核心诉求 | 典型行为 |
| --- | --- | --- |
| 项目经理 | 快速掌握项目风险、延期、阻塞 | 查看项目风险周报、跟进高风险事项 |
| 产品经理 | 从客户反馈中识别需求和优先级 | 分析反馈、合并相似问题、转需求 |
| 测试人员 | 提高 Bug 提交质量、识别重复 Bug | 创建/查看 Bug，使用 AI 分析相似问题 |
| 开发人员 | 快速理解 Bug 和文档上下文 | 查看 AI 摘要、相似 Bug、日志分析 |
| 客服/客户成功 | 把客户反馈转成研发可处理事项 | 提交反馈、查看分类和处理建议 |
| 管理者 | 查看项目整体健康度和客户问题趋势 | 查看项目周报和客户反馈 Top 问题 |

## 4. 业务流程

### 4.1 Bug AI 分析流程

```text
用户打开 Bug 详情页
  ↓
点击“AI 分析”
  ↓
系统展示分析中状态
  ↓
AI 返回分析结果
  ↓
用户查看相似 Bug、优先级建议、风险说明
  ↓
用户选择采纳或驳回
  ↓
采纳后写入 Bug 评论或 AI 分析记录
```

### 4.2 客户反馈分类流程

```text
客户反馈进入禅道
  ↓
用户打开反馈详情页
  ↓
点击“AI 分类”
  ↓
AI 生成分类、摘要、严重程度、相似反馈
  ↓
用户确认是否转 Bug/需求
  ↓
系统生成 Bug/需求草稿
  ↓
用户确认创建
  ↓
反馈与新 Bug/需求建立关联
```

### 4.3 文档问答流程

```text
用户进入文档详情页或项目知识问答页
  ↓
输入问题
  ↓
系统按权限检索相关文档、需求、Bug、任务
  ↓
AI 生成回答
  ↓
展示答案和引用来源
  ↓
用户可复制、追问或跳转来源
```

### 4.4 项目风险周报流程

```text
系统每周定时生成项目风险周报
  ↓
统计延期任务、阻塞任务、严重 Bug、客户反馈
  ↓
AI 总结风险和建议
  ↓
项目经理查看周报
  ↓
项目经理确认、编辑或发布
```

## 5. 页面需求

### 5.1 Bug 详情页 AI 面板

入口位置：

- Bug 详情页操作区增加“AI 分析”按钮。
- Bug 详情页右侧或正文下方展示“AI 分析结果”面板。

功能：

- 触发 AI 分析。
- 展示分析状态。
- 展示分析结果。
- 支持采纳、驳回、重新分析。
- 支持查看历史分析。

展示字段：

| 字段 | 说明 |
| --- | --- |
| 问题摘要 | AI 对 Bug 的简要总结 |
| 建议优先级 | P0/P1/P2/P3 |
| 建议严重程度 | 阻塞/严重/一般/轻微 |
| 相似 Bug | 显示 Bug ID、标题、相似度、状态 |
| 风险说明 | 对版本、模块、客户影响的说明 |
| 建议处理人 | 可选，显示推荐理由 |
| 证据来源 | 引用相似 Bug、文档、反馈 |
| 置信度 | 0-100% |

操作：

- `AI 分析`
- `重新分析`
- `采纳建议`
- `驳回`
- `写入评论`
- `查看相似 Bug`

验收标准：

- 用户可在 Bug 详情页看到 AI 分析入口。
- 点击后能返回结构化分析结果。
- 相似 Bug 可点击跳转。
- 采纳后能写入 Bug 评论。
- 无权限数据不能出现在分析结果中。

### 5.2 客户反馈详情页 AI 面板

入口位置：

- 反馈详情页增加“AI 分类”按钮。
- 页面展示“AI 分类结果”面板。

功能：

- 自动判断反馈类型。
- 自动生成摘要。
- 推荐严重程度和优先级。
- 推荐关联已有 Bug/需求。
- 支持生成 Bug/需求草稿。

反馈分类：

```text
Bug
需求
咨询
投诉
环境问题
使用问题
其他
```

展示字段：

| 字段 | 说明 |
| --- | --- |
| 反馈摘要 | 提炼客户问题 |
| 反馈类型 | Bug/需求/咨询等 |
| 建议优先级 | P0/P1/P2/P3 |
| 影响范围 | 单客户/多客户/核心流程 |
| 相似反馈 | 相似反馈列表 |
| 推荐关联 | 已有 Bug/需求 |
| 转换建议 | 建议转 Bug 或需求 |
| 证据来源 | 相似反馈、历史 Bug、文档 |

操作：

- `AI 分类`
- `转 Bug 草稿`
- `转需求草稿`
- `关联已有问题`
- `采纳`
- `驳回`

验收标准：

- 单条反馈可以被 AI 分类。
- AI 可生成 Bug/需求草稿。
- 用户确认后才创建正式 Bug/需求。
- 原反馈与新建对象保留关联。

### 5.3 文档详情页 AI 助手

入口位置：

- 文档详情页增加“AI 总结”和“提问”入口。
- 项目文档库增加“项目知识问答”入口。

功能：

- 单文档摘要。
- 单文档问答。
- 项目范围问答。
- 文档缺口检查。
- 展示引用来源。

常用问题模板：

```text
总结这篇文档
这篇需求缺少哪些验收标准？
根据文档生成测试点
这个功能涉及哪些模块？
有哪些未明确的边界条件？
```

展示字段：

| 字段 | 说明 |
| --- | --- |
| 回答内容 | AI 生成答案 |
| 引用来源 | 文档段落、需求、Bug、任务 |
| 置信度 | 可选 |
| 追问入口 | 继续提问 |

验收标准：

- 用户可对单文档提问。
- 用户可在项目范围内提问。
- 回答必须展示引用来源。
- 无权限文档不能被检索和引用。

### 5.4 项目风险周报页

入口位置：

- 项目概览页增加“AI 风险周报”入口。
- 项目菜单增加“AI 报告”页。

报告内容：

| 区块 | 内容 |
| --- | --- |
| 本周进展 | 已完成需求、任务、Bug |
| 主要风险 | 延期、阻塞、严重 Bug、客户投诉 |
| 质量情况 | 新增 Bug、关闭 Bug、重开 Bug、严重 Bug |
| 客户反馈 | Top 问题、重点客户、重复反馈 |
| AI 建议 | 下周关注事项、建议负责人、建议处理顺序 |
| 证据来源 | 任务、Bug、反馈、文档链接 |

操作：

- `生成周报`
- `重新生成`
- `编辑`
- `发布`
- `导出`

验收标准：

- 项目经理可生成指定项目的周报。
- 周报包含关键风险和证据链接。
- 用户可编辑 AI 生成内容。
- 发布前不会自动推送。

### 5.5 AI 分析历史页

功能：

- 查看 AI 对某个对象的历史分析。
- 查看采纳/驳回状态。
- 查看分析时间、发起人、模型、结果。

筛选条件：

- 对象类型。
- 项目。
- 分析类型。
- 状态。
- 发起人。
- 时间范围。

验收标准：

- 用户能查看自己有权限对象的 AI 分析历史。
- 管理员可查看全项目分析记录。
- 分析记录可追溯到原始对象。

## 6. 功能需求详情

### 6.1 Bug AI 分析

输入：

- Bug 标题。
- 复现步骤。
- 实际结果。
- 期望结果。
- 严重程度。
- 优先级。
- 所属项目/产品/模块。
- 评论摘要。
- 相关文档和相似 Bug。

输出：

```json
{
  "summary": "问题摘要",
  "suggested_priority": "P1",
  "suggested_severity": "严重",
  "risk_level": "high",
  "similar_bugs": [],
  "suggested_owner": "可选",
  "reason": "建议理由",
  "evidence": [],
  "confidence": 0.82
}
```

规则：

- 如果证据不足，应明确提示“信息不足”。
- 如果相似 Bug 已关闭，应提示是否可能重复。
- 如果相似 Bug 未关闭，应建议关联已有 Bug。
- AI 不自动修改原 Bug 字段。

### 6.2 客户反馈 AI 分类

输入：

- 反馈原文。
- 客户名称。
- 客户等级。
- 来源渠道。
- 关联产品/项目。
- 历史反馈。
- 相似 Bug/需求。

输出：

```json
{
  "category": "Bug",
  "summary": "客户在登录后无法进入首页",
  "impact_scope": "single_customer",
  "suggested_priority": "P1",
  "suggested_action": "create_bug",
  "similar_feedback": [],
  "related_objects": [],
  "draft": {
    "title": "登录后无法进入首页",
    "description": "..."
  }
}
```

规则：

- 反馈分类必须可人工修改。
- 转 Bug/需求前必须弹出草稿确认页。
- 不允许 AI 自动创建正式 Bug/需求。
- 重复反馈优先建议关联已有对象。

### 6.3 文档问答

输入：

- 用户问题。
- 当前文档或项目范围。
- 用户权限范围。
- 检索到的文档片段。

输出：

```json
{
  "answer": "回答内容",
  "sources": [
    {
      "object_type": "doc",
      "object_id": "88",
      "title": "登录模块 PRD",
      "snippet": "引用片段"
    }
  ],
  "confidence": 0.78
}
```

规则：

- 没有证据时必须回答“未在当前资料中找到依据”。
- 回答必须显示引用来源。
- 不能引用用户无权限访问的文档。
- 支持追问。

### 6.4 项目风险周报

输入：

- 项目本周任务。
- 项目本周 Bug。
- 项目本周客户反馈。
- 项目版本计划。
- 项目文档和会议纪要。

输出：

```json
{
  "summary": "本周项目总体风险中等",
  "progress": [],
  "risks": [],
  "quality": [],
  "customer_feedback": [],
  "suggestions": [],
  "evidence": []
}
```

规则：

- 风险必须关联具体对象。
- 周报可编辑。
- 周报发布前必须人工确认。
- 周报生成后保留历史版本。

## 7. 状态与交互规则

### 7.1 AI 分析状态

```text
未分析
分析中
分析成功
分析失败
待确认
已采纳
已驳回
已写回
```

### 7.2 人工确认规则

AI 结果默认只作为建议。

需要人工确认的操作：

- 写入评论。
- 修改优先级。
- 修改严重程度。
- 创建 Bug。
- 创建需求。
- 关联已有 Bug/需求。
- 发布周报。

### 7.3 异常状态

| 场景 | 页面提示 |
| --- | --- |
| AI 服务不可用 | AI 服务暂不可用，请稍后重试 |
| 分析超时 | 分析任务仍在处理中，可稍后查看 |
| 权限不足 | 无权分析该对象 |
| 文档未索引 | 文档正在索引，请稍后再试 |
| 证据不足 | 当前信息不足，无法生成可靠建议 |

## 8. 权限需求

### 8.1 权限原则

- 用户只能分析自己有权访问的禅道对象。
- AI 问答只能检索用户有权限访问的数据。
- AI 分析结果继承原对象权限。
- 管理员可配置哪些项目启用 AI。
- 管理员可配置哪些角色能使用 AI。

### 8.2 角色权限

| 操作 | 管理员 | 项目经理 | 产品经理 | 测试 | 开发 | 客服 |
| --- | --- | --- | --- | --- | --- | --- |
| 使用 Bug AI | 是 | 是 | 是 | 是 | 是 | 否 |
| 使用反馈 AI | 是 | 是 | 是 | 是 | 否 | 是 |
| 使用文档问答 | 是 | 是 | 是 | 是 | 是 | 部分 |
| 生成项目周报 | 是 | 是 | 否 | 否 | 否 | 否 |
| 发布项目周报 | 是 | 是 | 否 | 否 | 否 | 否 |
| 配置 AI 服务 | 是 | 否 | 否 | 否 | 否 | 否 |

## 11. 非功能需求

### 11.1 性能

- 单条 Bug AI 分析：同步等待不超过 10 秒；超过后转后台任务。
- 文档问答：常规问题 15 秒内返回。
- 项目周报：允许后台生成，5 分钟内完成。
- 向量同步延迟：普通对象 5 分钟内完成。

### 11.2 可用性

- AI 服务不可用时，不影响禅道原有功能。
- AI 面板加载失败时，页面其他内容正常显示。
- 分析失败可重试。

### 11.3 安全

- AI 不可绕过禅道权限。
- 外部模型调用支持脱敏。
- API Key 不在前端暴露。
- 所有 AI 写回动作记录审计日志。

### 11.4 可解释性

- AI 输出必须包含理由。
- 涉及判断的结论必须包含证据来源。
- 文档问答必须展示引用来源。

## 12. 埋点与指标

### 12.1 使用指标

- AI 分析按钮点击次数。
- Bug AI 分析次数。
- 反馈 AI 分类次数。
- 文档问答次数。
- 周报生成次数。

### 12.2 质量指标

- AI 建议采纳率。
- AI 建议驳回率。
- 相似 Bug 命中率。
- 反馈分类人工修改率。
- 文档问答有用率。

### 12.3 效率指标

- 反馈转 Bug 平均耗时。
- Bug 去重数量。
- 周报人工编辑耗时。
- 文档查找耗时下降情况。

## 13. MVP 验收标准

MVP 完成标准：

1. 禅道 Bug 详情页能显示 AI 分析入口。
2. Bug AI 能返回相似 Bug、优先级建议、风险说明。
3. 客户反馈详情页能进行 AI 分类。
4. 客户反馈能生成 Bug/需求草稿。
5. 文档详情页支持 AI 总结和问答。
6. 项目维度能生成 AI 风险周报。
7. 所有 AI 结果默认不自动写回，必须人工确认。
8. AI 结果能展示证据来源。
9. 用户无权限的数据不会出现在 AI 回答中。
10. AI 服务异常不影响禅道原有功能。
