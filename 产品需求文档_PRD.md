# 基于禅道的 AI 项目管理助手 PRD

版本：v1.2
日期：2026-07-10
产品名称：禅道 AI 项目管理助手  
文档类型：产品需求文档 PRD  
首批 MVP 范围：基础 AI 服务、Bug AI 分析、单文档摘要与问答、全局 AI 客服聊天、禅道扩展入口、对象级权限校验与人工确认机制

## 1. 产品概述

### 1.1 产品定位

禅道 AI 项目管理助手是一套基于禅道开源版的 AI 增强系统，通过禅道扩展页面接入独立 AI 服务，帮助研发团队分析 Bug、客户反馈、项目文档和项目风险。

产品不替代禅道，而是增强禅道：

- 禅道负责项目、需求、任务、Bug、测试、文档、反馈等基础管理。
- AI 助手负责分析、总结、归类、检索、生成建议。
- 用户确认后，AI 建议才能写回禅道。

### 1.2 产品目标

MVP 目标：

- 降低 Bug 分析和重复排查成本。
- 让单篇项目文档可问答、可总结，并展示引用来源。
- 让用户从任意业务页面快速打开 AI 客服，获得平台使用帮助或当前 Bug/文档上下文回答。
- 验证 Bug、文档的同步、向量化、权限过滤和 AI 分析闭环。
- 验证“禅道扩展 + 独立 AI 服务”的产品可行性。

### 1.3 非目标

MVP 不做：

- 不替代禅道原有项目管理流程。
- 不自动修改 Bug 状态、负责人、优先级。
- 不自动关闭问题。
- 不做完整 BI 报表平台。
- 不做移动端。
- 不做复杂多租户商业后台。
- 不做客户反馈分类、项目风险周报和项目范围知识问答，这些能力进入 V1.1。
- 不做模型管理后台；首批模型参数通过服务端配置注入。
- 不写回自定义字段，首批仅允许用户确认后添加评论。

## 2. 用户角色

| 角色 | 核心诉求 | 典型行为 |
| --- | --- | --- |
| 项目经理 | 快速掌握项目风险、延期、阻塞 | 查看项目风险周报、跟进高风险事项 |
| 产品经理 | 从客户反馈中识别需求和优先级 | 分析反馈、合并相似问题、转需求 |
| 测试人员 | 提高 Bug 提交质量、识别重复 Bug | 创建/查看 Bug，使用 AI 分析相似问题 |
| 开发人员 | 快速理解 Bug 和文档上下文 | 查看 AI 摘要、相似 Bug、日志分析 |
| 客服/客户成功 | 把客户反馈转成研发可处理事项 | 提交反馈、查看分类和处理建议 |
| 管理者 | 查看项目整体健康度和客户问题趋势 | 查看项目周报和客户反馈 Top 问题 |

## 3. 首批 MVP 功能范围

| 模块 | 功能 | 优先级 |

| 文档问答 | 单文档摘要、单文档问答、引用来源 | P0 |

## 4. 业务流程

### 4.1 Bug AI 分析流程

```text
用户打开 Bug 详情页
  ↓
点击“AI 分析”
  ↓
系统展示分析中状态
  ↓
AI 返回分析结果
  ↓
用户查看相似 Bug、优先级建议、风险说明
  ↓
用户选择采纳或驳回
  ↓
采纳后写入 Bug 评论或 AI 分析记录
```

### 4.2 客户反馈分类流程（V1.1）

```text
客户反馈进入禅道
  ↓
用户打开反馈详情页
  ↓
点击“AI 分类”
  ↓
AI 生成分类、摘要、严重程度、相似反馈
  ↓
用户确认是否转 Bug/需求
  ↓
系统生成 Bug/需求草稿
  ↓
用户确认创建
  ↓
反馈与新 Bug/需求建立关联
```

### 4.3 文档问答流程

```text
用户进入文档详情页
  ↓
输入问题
  ↓
系统在当前文档内按权限检索相关片段
  ↓
AI 生成回答
  ↓
展示答案和引用来源
  ↓
用户可复制、追问或跳转来源
```

### 4.4 项目风险周报流程（V1.1）

```text
系统每周定时生成项目风险周报
  ↓
统计延期任务、阻塞任务、严重 Bug、客户反馈
  ↓
AI 总结风险和建议
  ↓
项目经理查看周报
  ↓
项目经理确认、编辑或发布
```

### 4.5 全局 AI 客服聊天流程

```text
用户在任意已登录业务页面点击右下角机器人图标
  ↓
系统打开 AI 客服弹窗并识别当前页面上下文
  ↓
Bug/文档页绑定当前对象；其他页面进入平台使用帮助范围
  ↓
用户提问并获得流式回答
  ↓
回答展示引用、无证据提示或权限提示
  ↓
用户可继续追问、查看历史会话或关闭弹窗继续原页面操作
```

## 5. 页面需求

### 5.1 Bug 详情页 AI 面板

入口位置：

- Bug 详情页操作区增加“AI 分析”按钮。
- Bug 详情页右侧或正文下方展示“AI 分析结果”面板。

功能：

- 触发 AI 分析。
- 展示分析状态。
- 展示分析结果。
- 支持采纳、驳回、重新分析。
- 支持查看历史分析。

展示字段：

| 字段 | 说明 |
| --- | --- |
| 问题摘要 | AI 对 Bug 的简要总结 |
| 建议优先级 | P0/P1/P2/P3 |
| 建议严重程度 | 阻塞/严重/一般/轻微 |
| 相似 Bug | 显示 Bug ID、标题、相似度、状态 |
| 风险说明 | 对版本、模块、客户影响的说明 |
| 建议处理人 | 可选，显示推荐理由 |
| 证据来源 | 引用相似 Bug、文档、反馈 |
| 置信度 | 0-100% |

操作：

- `AI 分析`
- `重新分析`
- `采纳建议`
- `驳回`
- `写入评论`
- `查看相似 Bug`

验收标准：

- 用户可在 Bug 详情页看到 AI 分析入口。
- 点击后能返回结构化分析结果。
- 相似 Bug 可点击跳转。
- 采纳后能写入 Bug 评论。
- 无权限数据不能出现在分析结果中。

### 5.2 客户反馈详情页 AI 面板（V1.1）

入口位置：

- 反馈详情页增加“AI 分类”按钮。
- 页面展示“AI 分类结果”面板。

功能：

- 自动判断反馈类型。
- 自动生成摘要。
- 推荐严重程度和优先级。
- 推荐关联已有 Bug/需求。
- 支持生成 Bug/需求草稿。

反馈分类：

```text
Bug
需求
咨询
投诉
环境问题
使用问题
其他
```

展示字段：

| 字段 | 说明 |
| --- | --- |
| 反馈摘要 | 提炼客户问题 |
| 反馈类型 | Bug/需求/咨询等 |
| 建议优先级 | P0/P1/P2/P3 |
| 影响范围 | 单客户/多客户/核心流程 |
| 相似反馈 | 相似反馈列表 |
| 推荐关联 | 已有 Bug/需求 |
| 转换建议 | 建议转 Bug 或需求 |
| 证据来源 | 相似反馈、历史 Bug、文档 |

操作：

- `AI 分类`
- `转 Bug 草稿`
- `转需求草稿`
- `关联已有问题`
- `采纳`
- `驳回`

验收标准：

- 单条反馈可以被 AI 分类。
- AI 可生成 Bug/需求草稿。
- 用户确认后才创建正式 Bug/需求。
- 原反馈与新建对象保留关联。

### 5.3 文档详情页 AI 助手

入口位置：

- 文档详情页增加“AI 总结”和“提问”入口。
- V1.1 在项目文档库增加“项目知识问答”入口。

功能：

- 单文档摘要。
- 单文档问答。
- 展示引用来源。
- V1.1 增加项目范围问答和文档缺口检查。

常用问题模板：

```text
总结这篇文档
这篇需求缺少哪些验收标准？
根据文档生成测试点
这个功能涉及哪些模块？
有哪些未明确的边界条件？
```

展示字段：

| 字段 | 说明 |
| --- | --- |
| 回答内容 | AI 生成答案 |
| 引用来源 | 文档段落、需求、Bug、任务 |
| 置信度 | 可选 |
| 追问入口 | 继续提问 |

验收标准：

- 用户可对单文档提问。
- 回答必须展示引用来源。
- 无权限文档不能被检索和引用。

### 5.4 项目风险周报页（V1.1）

入口位置：

- 项目概览页增加“AI 风险周报”入口。
- 项目菜单增加“AI 报告”页。

报告内容：

| 区块 | 内容 |
| --- | --- |
| 本周进展 | 已完成需求、任务、Bug |
| 主要风险 | 延期、阻塞、严重 Bug、客户投诉 |
| 质量情况 | 新增 Bug、关闭 Bug、重开 Bug、严重 Bug |
| 客户反馈 | Top 问题、重点客户、重复反馈 |
| AI 建议 | 下周关注事项、建议负责人、建议处理顺序 |
| 证据来源 | 任务、Bug、反馈、文档链接 |

操作：

- `生成周报`
- `重新生成`
- `编辑`
- `发布`
- `导出`

验收标准：

- 项目经理可生成指定项目的周报。
- 周报包含关键风险和证据链接。
- 用户可编辑 AI 生成内容。
- 发布前不会自动推送。

功能：

- 查看采纳/驳回状态。
- 查看分析时间、发起人、模型、结果。

筛选条件：

- 对象类型。
- 项目。
- 分析类型。
- 状态。
- 发起人。
- 时间范围。

验收标准：

- 用户能查看自己有权限对象的 AI 分析历史。

### 5.6 AI 服务配置页（V1.1）

首批 MVP 只通过 AI 服务端环境变量或受保护配置文件注入模型参数，不开发模型管理页面。以下页面能力在 V1.1 实现。

功能：

- 配置 AI 服务地址和服务鉴权 Token。
- 配置默认聊天模型、默认 Embedding 模型。
- 配置是否允许调用公网模型。
- 配置模型调用超时时间、重试次数、并发限制。
- 配置敏感信息脱敏规则。
- 查看 AI 服务健康状态。

配置字段：

| 字段 | 说明 |
| --- | --- |
| AI 服务地址 | 独立 AI 服务 API 地址 |
| 服务 Token | 禅道扩展调用 AI 服务的鉴权 Token |
| 默认聊天模型 | Bug 分析、反馈分类、周报生成使用的默认模型 |
| 默认 Embedding 模型 | 向量化使用的模型 |
| 调用超时 | 单次模型调用超时时间 |
| 最大重试次数 | 模型调用失败后的重试次数 |
| 公网模型开关 | 是否允许调用外部公网模型 |
| 脱敏开关 | 是否在外部模型调用前脱敏 |

验收标准：

- 只有管理员可以配置 AI 服务。
- API Key、Token 不在前端明文展示。
- 配置保存后可进行连接测试。
- AI 服务不可用时给出明确错误提示。

### 5.7 索引与任务状态页

功能：

- 查看对象同步状态。
- 查看文档解析状态。
- 查看向量化状态。
- 查看异步分析任务状态。
- 支持失败任务重试。

状态：

```text
待同步
同步中
已同步
解析中
解析失败
向量化中
已索引
索引失败
任务排队中
任务执行中
任务失败
任务完成
```

验收标准：

- 用户能知道文档问答不可用是因为未索引、索引中还是索引失败。
- 管理员可查看失败原因并手动重试。
- 普通用户只能查看自己有权限对象的索引状态。

### 5.8 全局 AI 客服悬浮入口与聊天弹窗

入口和布局：

- 登录后的业务页面右下角固定机器人图标；登录、安装、升级和公开页面不展示。
- 图标不得遮挡页面主要操作、表格分页、保存按钮或移动端安全区域。
- 点击后打开右侧弹窗或抽屉，桌面端建议宽度 `400-480px`；关闭后不刷新当前页面。
- 弹窗支持最小化、关闭、重新打开，并保留当前会话状态。

聊天能力：

- 展示欢迎语、快捷问题、消息列表、输入框、发送/停止生成和清空当前会话。
- 支持流式输出、连续追问、引用来源、复制答案和有用/无用反馈。
- Bug 页面自动创建 `bug` 范围会话；文档页面自动创建 `doc` 范围会话。
- 其他页面使用 `help` 范围，只检索经过审核的平台使用手册、FAQ 和操作说明，不检索项目业务数据。
- 同一页面重新打开时恢复最近会话；切换到不同 Bug 或文档时创建或切换到对应范围会话。
- AI 服务不可用时显示失败提示和重试入口，不影响当前禅道页面。

安全规则：

- 禅道扩展根据当前路由生成受信任的页面上下文，浏览器不能通过请求正文扩大对象范围。
- 每次发送消息、读取历史消息和返回引用时重新校验当前用户权限。
- 弹窗不能接收客户端自定义 System Prompt、模型密钥或任意检索范围。
- `help` 范围与项目业务知识库隔离，不能用于绕过 Bug、文档对象权限。

验收标准：

- 用户可从任意已登录业务页面打开和关闭 AI 客服弹窗。
- Bug、文档页面的回答不会超出当前绑定对象允许的检索范围。
- 非 Bug/文档页面可以回答平台使用问题，但不会返回项目业务内容。
- 流式中断后可通过历史消息恢复最终状态。

## 6. 功能需求详情

### 6.1 Bug AI 分析

输入：

- Bug 标题。
- 复现步骤。
- 实际结果。
- 期望结果。
- 严重程度。
- 优先级。
- 所属项目/产品/模块。
- 评论摘要。
- 附件和截图的 OCR 文本。
- 截图的 AI 图片描述。
- 相关文档和相似 Bug。

输出：

```json
{
  "summary": "问题摘要",
  "suggested_priority": "P1",
  "suggested_severity": "严重",
  "risk_level": "high",
  "similar_bugs": [],
  "suggested_owner": "可选",
  "reason": "建议理由",
  "evidence": [],
  "confidence": 0.82
}
```

规则：

- 如果证据不足，应明确提示“信息不足”。
- 如果相似 Bug 已关闭，应提示是否可能重复。
- 如果相似 Bug 未关闭，应建议关联已有 Bug。
- AI 不自动修改原 Bug 字段。
- Bug 创建页可提供质量检查，提示标题不清晰、复现步骤缺失、实际结果和期望结果缺失、环境信息不足等问题。
- Bug 创建页可在提交前提示疑似重复 Bug，但不得阻止用户提交。

### 6.2 客户反馈 AI 分类（V1.1）

输入：

- 反馈原文。
- 客户名称。
- 客户等级。
- 来源渠道。
- 关联产品/项目。
- 历史反馈。
- 相似 Bug/需求。

输出：

```json
{
  "category": "Bug",
  "summary": "客户在登录后无法进入首页",
  "impact_scope": "single_customer",
  "suggested_priority": "P1",
  "suggested_action": "create_bug",
  "similar_feedback": [],
  "related_objects": [],
  "draft": {
    "title": "登录后无法进入首页",
    "description": "..."
  }
}
```

规则：

- 反馈分类必须可人工修改。
- 转 Bug/需求前必须弹出草稿确认页。
- 不允许 AI 自动创建正式 Bug/需求。
- 重复反馈优先建议关联已有对象。

### 6.3 文档问答

输入：

- 用户问题。
- 当前文档；项目范围检索在 V1.1 支持。
- 用户权限范围。
- 检索到的文档片段。
- V1.1 可增加文档图片 OCR 文本和图片描述。

输出：

```json
{
  "answer": "回答内容",
  "sources": [
    {
      "object_type": "doc",
      "object_id": "88",
      "title": "登录模块 PRD",
      "snippet": "引用片段"
    }
  ],
  "confidence": 0.78
}
```

规则：

- 没有证据时必须回答“未在当前资料中找到依据”。
- 回答必须显示引用来源。
- 不能引用用户无权限访问的文档。
- 单文档快捷问答与全局 AI 客服复用同一套会话历史、权限和引用实现。
- V1.1 引用图片时必须展示图片所在文档、章节和图片链接；OCR 置信度低时提示“图片文字识别可能不准确”。

### 6.4 项目风险周报（V1.1）

输入：

- 项目本周任务。
- 项目本周 Bug。
- 项目本周客户反馈。
- 项目版本计划。
- 项目文档和会议纪要。

输出：

```json
{
  "summary": "本周项目总体风险中等",
  "progress": [],
  "risks": [],
  "quality": [],
  "customer_feedback": [],
  "suggestions": [],
  "evidence": []
}
```

规则：

- 风险必须关联具体对象。
- 周报可编辑。
- 周报发布前必须人工确认。
- 周报生成后保留历史版本。

### 6.5 全局 AI 客服聊天

输入：

- 用户消息。
- 由禅道服务端签名确认的当前页面类型、对象类型和对象 ID。
- Conversation 固定检索范围和最近历史消息。

输出：

- 流式回答文本。
- 已通过权限校验的引用来源。
- `insufficient_evidence`、任务状态和用量摘要。

规则：

- 首批支持 `help`、`bug`、`doc` 三种会话范围。
- 多轮追问不能扩大创建会话时的范围。
- 页面关闭或网络中断不代表消息失败，客户端通过历史接口恢复最终结果。
- 历史消息遵守租户保留策略；用户删除会话后进入异步清除流程。

## 8. 权限需求

- 用户只能分析自己有权访问的禅道对象。

- AI 分析结果继承原对象权限。
- 管理员可配置哪些项目启用 AI。
- 管理员可配置哪些角色能使用 AI。

## 9. 数据需求

### 9.1 需要同步的数据

首批 MVP 只同步下表中的 Bug 和文档文本字段。其余对象和附件/图片处理属于 V1.1 或后续版本。

| 对象 | 字段 |
| --- | --- |
| Bug | 标题、复现步骤、期望结果、实际结果、严重程度、优先级、状态、模块、版本、评论 |
| 需求 | 标题、描述、验收标准、优先级、状态、版本 |
| 任务 | 标题、描述、负责人、状态、预计/消耗/剩余工时、截止日期 |
| 文档 | 标题、正文、目录、附件文本、图片 OCR、图片描述、更新时间 |
| 客户反馈 | 原文、客户、来源、处理状态、关联对象 |
| 测试用例 | 标题、前置条件、步骤、预期结果、执行结果 |
| 项目 | 名称、成员、计划时间、实际进度、里程碑 |
| 附件/图片 | 文件 ID、文件名、文件类型、OCR 文本、图片描述、所属对象 |

### 9.2 向量化数据

首批 MVP 需要向量化：

- Bug 标题和复现步骤。
- Bug 实际结果、期望结果、关键评论。
- 文档正文。

V1.1 或后续版本再向量化 Bug/文档图片 OCR、客户反馈、需求和测试用例。

不建议向量化：

- 负责人、状态、优先级、严重程度、时间等结构化字段。
- 全量操作日志。
- 低价值评论，例如“收到”“已处理”“+1”。
- 权限无法确认的数据。

分段规则：

| 对象 | 分段方式 | 建议大小 |
| --- | --- | --- |
| Bug | 单条 Bug 作为一个 chunk，评论多时关键评论单独切片 | 300-1200 字 |
| 客户反馈 | 单条反馈作为一个 chunk | 100-800 字 |
| 需求 | 单条需求作为一个 chunk，长 PRD 按章节切片 | 300-1200 字 |
| 测试用例 | 单条用例作为一个 chunk | 200-800 字 |
| 文档 | 按 H1/H2/H3 和段落切片 | 500-1000 字 |
| 会议纪要 | 按议题或段落切片 | 500-1000 字 |
| 图片 | OCR 文本 + 图片描述 + 所属对象上下文作为 image chunk | 100-800 字 |

文档切片要求：

- 优先按标题层级切片。
- 章节超过 1000 中文字时按段落继续切分。
- chunk 之间保留 100-150 字 overlap。
- 每个 chunk 必须保留文档标题、章节路径、对象 ID 和来源链接。

图片处理要求（V1.1）：

- 图片原文件不存入向量库。
- 向量库只保存 OCR 文本、图片描述、图片来源信息。
- Bug 和文档图片作为独立 `image` chunk。
- 图片 chunk 必须继承原 Bug 或文档权限。
- 含手机号、邮箱、Token、Cookie 等敏感信息的截图必须先脱敏。

### 9.3 AI 分析记录

每条 AI 分析记录至少包含：

- 对象类型。
- 对象 ID。
- 分析类型。
- 分析结果。
- 证据来源。
- 置信度。
- 状态。
- 发起人。
- 发起时间。
- 采纳/驳回人。
- 采纳/驳回时间。
- 模型名称。
- Prompt 版本。
- request_id。
- task_id。
- 输入摘要。
- token 消耗。
- 耗时。
- 错误码和错误信息。

### 9.4 向量数据结构

每个向量点对应一个文本 chunk 或 image chunk。

```json
{
  "id": "bug:123:chunk:1",
  "object_type": "bug",
  "object_id": "123",
  "chunk_type": "text",
  "chunk_index": 1,
  "title": "登录后无法进入首页",
  "content": "标题：登录后无法进入首页\n复现步骤：...\n实际结果：...\n期望结果：...",
  "project_id": "8",
  "product_id": "3",
  "module_id": "12",
  "status": "active",
  "permission_scope": ["project:8", "product:3"],
```

## 11. 非功能需求

### 11.1 性能

- 单条 Bug AI 分析：同步等待不超过 10 秒；超过后转后台任务。
- 文档问答：常规问题 15 秒内返回。
- AI 客服：弹窗 1 秒内打开；首个流式文本目标 3 秒内返回，模型不可用时 10 秒内明确失败。
- V1.1 项目周报：允许后台生成，5 分钟内完成。
- 向量同步延迟：普通对象 5 分钟内完成。

### 11.2 可用性

- AI 服务不可用时，不影响禅道原有功能。
- AI 面板加载失败时，页面其他内容正常显示。
- 分析失败可重试。

### 11.3 安全

- AI 不可绕过禅道权限。
- 外部模型调用支持脱敏。
- API Key 不在前端暴露。
- 所有 AI 写回动作记录审计日志。

### 11.4 可解释性

- AI 输出必须包含理由。
- 涉及判断的结论必须包含证据来源。
- 文档问答必须展示引用来源。

### 11.5 AI 质量评价

- AI 分析结果支持用户标记“有用/无用”。
- 用户可选择问题类型：分类错误、相似对象不准确、引用错误、建议不可执行、内容不完整。
- 用户修改 AI 草稿后，系统记录修改后的最终结果。
- AI 质量反馈用于后续优化 Prompt 和检索策略。

## 12. 埋点与指标

### 12.1 使用指标

- AI 分析按钮点击次数。
- Bug AI 分析次数。
- 反馈 AI 分类次数。
- 文档问答次数。

### 12.3 效率指标

- Bug 去重数量。

## 13. MVP 验收标准

MVP 完成标准：

1. 禅道 Bug 详情页能显示 AI 分析入口。
2. Bug AI 能返回相似 Bug、优先级建议、风险说明。
3. 文档详情页支持单文档 AI 总结和问答。
4. AI 结果能展示证据来源；无证据时明确说明资料不足。
5. AI 结果默认不自动写回；用户确认后只允许添加评论。
6. 禅道请求通过 HMAC 鉴权，重复 Nonce、跨租户和伪造身份请求被拒绝。
7. AI 服务通过禅道批量权限接口逐个校验候选对象，请求正文中的 `permission_scope` 不作为授权依据。
8. 用户无权限的数据不会进入模型上下文、回答或历史结果。
9. 对象重复同步、乱序更新和删除场景不会产生错误的有效索引。
10. AI 服务异常不影响禅道原有功能。
11. 登录后的业务页面右下角展示机器人图标，点击后可以在不离开当前页面的情况下聊天。
12. AI 客服支持 `help`、`bug`、`doc` 范围和历史消息；连续追问不会扩大会话范围。
13. 用户失去源对象权限后，相关历史回答和引用按 `restricted` 规则隐藏。
