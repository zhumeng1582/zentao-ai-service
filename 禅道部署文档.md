# 禅道部署文档

版本：v1.0  
日期：2026-07-09  
文档类型：部署实施文档  
适用范围：禅道开源版、禅道扩展、反馈管理、后续接入 zentao-ai-service 的 AI 能力

## 1. 部署目标

本文档用于指导禅道系统的本地开发、测试环境和生产环境部署，明确部署前需要准备的服务器资源、软件依赖、网络端口、数据库、存储、备份、安全和验收标准。

本项目的部署分为两个层级：

- 基础禅道部署：只部署禅道本体，支持项目、产品、需求、任务、Bug、测试、文档、反馈等基础功能。
- 禅道 + AI 服务部署：在禅道基础上增加独立 AI 服务、向量数据库、任务队列和模型调用能力。

## 2. 总体架构

### 2.1 基础禅道架构

```text
用户浏览器
  ↓
Nginx / Apache
  ↓
PHP-FPM / PHP Runtime
  ↓
禅道应用 zentaopms
  ↓
MySQL
```

### 2.2 禅道 + AI 服务架构

```text
用户浏览器
  ↓
Nginx / Apache
  ↓
禅道 zentaopms
  ↓ API / Webhook / 扩展调用
zentao-ai-service
  ├── ai-api
  ├── ai-worker
  ├── Redis
  ├── PostgreSQL
  ├── Qdrant
  └── LLM / Embedding 模型服务
```

部署原则：

- 禅道负责项目管理主业务和权限体系。
- AI 服务独立部署，不把大模型调用、向量库和异步任务放进禅道 PHP 应用。
- 禅道和 AI 服务之间通过内网 API 通信。
- 生产环境数据库、附件和向量库都必须纳入备份。

## 3. 资源需求

### 3.1 开发环境最低资源

适用于本地开发、功能验证、单人调试。

| 组件 | CPU | 内存 | 磁盘 | 说明 |
| --- | --- | --- | --- | --- |
| 禅道应用 | 2 核 | 2 GB | 10 GB | PHP 内置服务或 Nginx + PHP-FPM |
| MySQL | 2 核 | 2 GB | 20 GB | 可与禅道同机 |
| AI API | 2 核 | 2 GB | 5 GB | 仅调试接口时可选 |
| Redis | 1 核 | 512 MB | 1 GB | AI 异步任务可选 |
| Qdrant | 2 核 | 2 GB | 20 GB | 文档和 Bug 向量检索可选 |

开发环境建议总资源：

```text
CPU：4 核
内存：8 GB
磁盘：80 GB SSD
```

### 3.2 测试环境推荐资源

| 组件 | CPU | 内存 | 磁盘 | 实例数 |
| --- | --- | --- | --- | --- |
| 禅道应用 | 2 核 | 4 GB | 20 GB | 1 |
| MySQL | 2 核 | 4 GB | 100 GB SSD | 1 |
| AI API | 2 核 | 4 GB | 20 GB | 1 |
| AI Worker | 2 核 | 4 GB | 20 GB | 1 |
| Redis | 1 核 | 1 GB | 10 GB | 1 |
| PostgreSQL | 2 核 | 4 GB | 100 GB SSD | 1 |
| Qdrant | 2 核 | 4 GB | 100 GB SSD | 1 |

测试环境建议总资源：

```text
CPU：8-12 核
内存：16-24 GB
磁盘：300 GB SSD
```

### 3.3 生产环境推荐资源

适用于 50-200 人团队、正式项目管理和 AI 检索分析。

| 组件 | CPU | 内存 | 磁盘 | 实例数 | 说明 |
| --- | --- | --- | --- | --- | --- |
| 禅道应用 | 4 核 | 8 GB | 50 GB | 1-2 | 高可用时部署 2 实例 |
| MySQL | 4-8 核 | 16 GB | 300 GB+ SSD | 1 主 1 备 | 保存禅道业务数据 |
| AI API | 4 核 | 8 GB | 50 GB | 2 | 可水平扩展 |
| AI Worker | 4-8 核 | 8-16 GB | 50 GB | 1-3 | 处理解析、OCR、向量化 |
| Redis | 2 核 | 4 GB | 50 GB | 1 主 1 备 | 队列和缓存 |
| PostgreSQL | 4 核 | 8-16 GB | 300 GB+ SSD | 1 主 1 备 | AI 服务业务库 |
| Qdrant | 4-8 核 | 16-32 GB | 500 GB+ SSD | 1-3 | 向量库，建议独立磁盘 |
| 对象存储/MinIO | 2-4 核 | 4-8 GB | 500 GB+ | 1-3 | 保存解析文件、图片、附件副本 |

生产环境建议总资源：

```text
CPU：24 核以上
内存：64 GB 以上
磁盘：1 TB SSD 起步
```

资源估算说明：

- 只部署禅道时，主要资源压力在 MySQL、附件存储和 PHP Web 请求。
- 接入 AI 后，主要资源压力在文档解析、OCR、Embedding、Qdrant 检索和模型调用。
- Bug、文档、反馈图片越多，Qdrant、对象存储和 Worker 资源需求越高。
- 如果模型私有化部署，还需要单独准备 GPU 服务器；如果调用第三方模型 API，则不需要本地 GPU。

## 4. 软件依赖

### 4.1 禅道基础依赖

| 软件 | 推荐版本 | 说明 |
| --- | --- | --- |
| Linux | Ubuntu 22.04 / CentOS 7+ / Rocky Linux 9 | 生产环境推荐 Linux |
| Nginx | 1.20+ | Web 入口，反向代理 |

| PHP | 8.0-8.3 | 需按禅道版本确认兼容范围 |
| MySQL | 5.7 / 8.0 | 禅道主库 |
| PHP 扩展 | mysqli、pdo_mysql、mbstring、curl、gd、zip、xml、json、openssl、fileinfo | 禅道运行依赖 |

### 4.2 AI 服务依赖

| 软件 | 推荐版本 | 说明 |
| --- | --- | --- |
| Python | 3.11+ | zentao-ai-service |
| FastAPI | 按项目依赖锁定 | AI API |
| Celery / RQ | 按项目依赖锁定 | 异步任务 |
| PostgreSQL | 14+ | AI 服务业务库 |
| Redis | 6+ | 队列和缓存 |
| Qdrant | 1.9+ | 向量数据库 |
| Tesseract / OCR 服务 | 按方案选择 | 图片文字识别 |
| MinIO / S3 | 可选 | 附件和解析产物存储 |

### 4.3 模型资源

模型调用有两种方式：

| 方式 | 资源需求 | 适用场景 |
| --- | --- | --- |
| 第三方模型 API | 不需要本地 GPU，需要公网或专线访问模型服务 | MVP、快速上线 |
| 私有化模型 | 需要 GPU 服务器、模型权重、推理服务 | 数据不能出内网、调用量大 |

第三方模型 API 需要准备：

- LLM API Key。
- Embedding API Key。
- 网络出口白名单。
- 调用额度和费用预算。
- 模型网关配置，便于后续切换模型供应商。

私有化模型建议单独评估：

```text
小模型推理：1 张 24 GB 显存 GPU 起步
中大模型推理：1-4 张 48/80 GB 显存 GPU
Embedding 模型：可 CPU 部署，数据量大时建议 GPU
```

## 5. 网络与端口

### 5.1 端口规划

| 服务 | 默认端口 | 对外暴露 | 说明 |
| --- | --- | --- | --- |
| Nginx HTTP | 80 | 是 | 可跳转 HTTPS |
| Nginx HTTPS | 443 | 是 | 生产环境必须启用 |
| 禅道 PHP 内置服务 | 8080 | 否 | 仅开发环境使用 |
| MySQL | 3306 | 否 | 仅内网访问 |
| AI API | 8000 | 否/按需 | 禅道内网调用 |
| Redis | 6379 | 否 | 内网访问 |
| PostgreSQL | 5432 | 否 | 内网访问 |
| Qdrant | 6333 | 否 | 内网访问 |
| MinIO API | 9000 | 否/按需 | 内网访问 |
| MinIO Console | 9001 | 否 | 管理入口 |

生产环境建议：

- 只暴露 `80/443`。
- 数据库、Redis、Qdrant、AI API 均只允许内网访问。
- 禅道调用 AI 服务使用内网域名，例如 `http://zentao-ai-api:8000`。

## 7. 部署方式

### 7.1 本地开发部署

适用于开发调试。

启动 MySQL 后，进入禅道目录：

```bash
cd /Users/zhangchao/Desktop/pm/zentaopms
php -S 127.0.0.1:8080 -t www
```

访问：

```text
http://127.0.0.1:8080
```

关闭开发服务：

```bash
lsof -tiTCP:8080 -sTCP:LISTEN | xargs kill
```

如果端口被占用，可以改用：

```bash
php -S 127.0.0.1:8081 -t www
```

### 7.2 Nginx + PHP-FPM 部署

推荐用于测试和生产环境。

目录建议：

```text
/opt/zentao/zentaopms
/data/zentao/mysql
/data/zentao/files
/data/zentao/backup
```

Nginx 示例：

```nginx
server {
    listen 80;
    server_name zentao.example.com;

    root /opt/zentao/zentaopms/www;
    index index.php index.html;

    client_max_body_size 100m;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location ~ \.php$ {
        include fastcgi_params;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        fastcgi_pass 127.0.0.1:9000;
        fastcgi_read_timeout 300;
    }
}
```

PHP-FPM 关键配置建议：

```ini
memory_limit = 512M
upload_max_filesize = 100M
post_max_size = 100M
max_execution_time = 300
max_input_vars = 5000
date.timezone = Asia/Shanghai
```

### 7.3 Docker Compose 部署建议

MVP 可使用 Docker Compose 管理禅道、数据库和 AI 组件：

```text
docker-compose.yml
  zentao
  mysql
  ai-api
  ai-worker
  postgres
  redis
  qdrant
  minio
```

建议挂载独立数据卷：

```text
./data/mysql
./data/postgres
./data/qdrant
./data/minio
./data/zentao-www-data
./logs
```

注意：

- 容器内不要写死公网地址，服务间使用 Compose service name。
- 数据库、Qdrant、MinIO 必须使用持久化 volume。
- 生产环境不建议把数据库密码写在 compose 文件明文中，应使用 `.env` 或密钥管理系统。

## 8. 数据库准备

### 8.1 禅道 MySQL

需要准备：

- 数据库名：`zentao`
- 用户名：按环境创建，例如 `zentao`
- 密码：生产环境使用强密码
- 字符集：`utf8mb4`
- 排序规则：`utf8mb4_general_ci` 或按禅道安装程序要求

示例：

```sql
CREATE DATABASE zentao DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
CREATE USER 'zentao'@'%' IDENTIFIED BY 'change_this_password';
GRANT ALL PRIVILEGES ON zentao.* TO 'zentao'@'%';
FLUSH PRIVILEGES;
```

### 8.2 AI 服务 PostgreSQL

建议数据库：

```text
数据库：zentao_ai
用户：zentao_ai
字符集：UTF8
```

保存内容：

- 同步对象状态。
- 索引任务状态。
- 分析任务状态。
- 模型调用日志。
- 用户采纳/驳回记录。
- 向量 chunk 元数据副本。

### 8.3 Qdrant

建议集合：

```text
zentao_knowledge
```

向量 payload 必须包含：

- `tenant_id`
- `object_type`
- `object_id`
- `chunk_id`
- `project_id`
- `product_id`
- `module_id`
- `source_url`
- `permission_scope`
- `updated_at`
- `deleted`

## 9. 配置项

### 9.1 禅道配置

需要确认：

- Web 根目录指向 `zentaopms/www`。
- `config/my.php` 或安装生成配置中数据库连接正确。
- `www/data` 可写。
- `tmp` 可写。
- 上传大小满足附件需求。
- 反馈模块、文档模块、Bug 模块权限已配置。

### 9.2 AI 服务配置

建议通过环境变量配置：

```text
AI_SERVICE_ENV=prod
AI_SERVICE_DATABASE_URL=postgresql://user:pass@postgres:5432/zentao_ai
AI_SERVICE_REDIS_URL=redis://redis:6379/0
AI_SERVICE_QDRANT_URL=http://qdrant:6333
AI_SERVICE_OBJECT_STORAGE_ENDPOINT=http://minio:9000
AI_SERVICE_LLM_PROVIDER=openai-compatible
AI_SERVICE_LLM_BASE_URL=https://model-provider.example.com/v1
AI_SERVICE_LLM_API_KEY=******
AI_SERVICE_EMBEDDING_MODEL=text-embedding-xxx
AI_SERVICE_WEBHOOK_TOKEN=******
```

禅道侧需要配置：

```text
AI_SERVICE_BASE_URL=http://ai-api:8000
AI_SERVICE_TOKEN=******
AI_SERVICE_TIMEOUT=30
AI_SERVICE_ENABLE_SYNC=true
```

## 10. 部署步骤

### 10.1 基础禅道部署步骤

1. 准备服务器、域名、数据库和存储目录。
2. 安装 Nginx/Apache、PHP、PHP 扩展、MySQL。
3. 上传或拉取 `zentaopms` 代码到部署目录。
4. 配置 Web 根目录为 `zentaopms/www`。
5. 设置目录权限：

```bash
chown -R www-data:www-data /opt/zentao/zentaopms
chmod -R 755 /opt/zentao/zentaopms
chmod -R 777 /opt/zentao/zentaopms/www/data
chmod -R 777 /opt/zentao/zentaopms/tmp
```

6. 创建 MySQL 数据库和用户。
7. 访问安装页面完成初始化。
8. 登录管理员账号，配置组织、用户、权限、产品、项目。
9. 验证 Bug、文档、反馈、附件上传功能。

### 10.2 AI 服务部署步骤

1. 准备 PostgreSQL、Redis、Qdrant、对象存储。
2. 部署 zentao-ai-service。
3. 配置数据库连接、Redis、Qdrant、模型网关和 Token。
4. 执行数据库迁移。
5. 启动 `ai-api`。
6. 启动 `ai-worker`。
7. 验证健康检查：

```text
GET /healthz
GET /readyz
```

8. 在禅道侧配置 AI 服务地址和 Token。
9. 创建或编辑 Bug、文档、反馈，验证同步任务。
10. 在 AI 服务查看对象解析、切片、向量化状态。

## 13. 安全要求

### 13.1 访问安全

- 生产环境必须启用 HTTPS。
- 管理员账号启用强密码。
- 禁止数据库、Redis、Qdrant 暴露公网。
- AI 服务只允许禅道服务器或内网网关访问。
- 禅道调用 AI 服务必须带 Token 或签名。

### 13.2 数据安全

- 模型调用前需评估数据是否允许出网。
- 客户敏感信息、账号、手机号、邮箱等应支持脱敏。
- AI 服务日志不得记录完整密钥。
- 向量 payload 不保存敏感原文，只保存必要检索字段和元数据。
- 附件和图片 OCR 结果按权限过滤。

### 13.3 权限要求

- 禅道内已有权限仍作为最终权限来源。
- AI 检索必须带当前用户身份。
- 用户只能检索自己有权限访问的产品、项目、文档、Bug、反馈。
- AI 分析结果写回禅道前必须经过用户确认。

## 14. 监控与告警

### 14.1 禅道监控

建议监控：

- Nginx 5xx 数量。
- PHP-FPM 进程数和慢请求。
- MySQL CPU、连接数、慢 SQL、磁盘空间。
- 附件目录磁盘使用率。
- 登录失败次数。

### 14.2 AI 服务监控

建议监控：

- AI API 请求量、错误率、延迟。
- Worker 队列积压。
- 文档解析失败率。
- OCR 失败率。
- Embedding 调用失败率。
- Qdrant 检索延迟。
- 模型调用耗时、费用、失败率。

### 14.3 告警阈值建议

| 指标 | 阈值 |
| --- | --- |
| 磁盘使用率 | 超过 80% 告警，超过 90% 严重 |
| MySQL 连接数 | 超过 80% 告警 |
| API 5xx | 5 分钟内持续出现告警 |
| Worker 队列积压 | 超过 1000 条或超过 30 分钟未消费 |
| Qdrant 查询延迟 | P95 超过 2 秒告警 |
| 模型调用失败率 | 5 分钟内超过 10% 告警 |

- Bug 创建/编辑后能同步到 AI 服务。
- 文档创建/编辑后能同步到 AI 服务。
- 图片附件能进入解析队列。
- Qdrant 中能看到对应向量 chunk。
- AI 检索结果遵守禅道权限。
- AI 分析结果写回前需要人工确认。

### 15.3 备份验收

- MySQL 备份可恢复。
- 附件备份可恢复。
- PostgreSQL 备份可恢复。
- Qdrant 快照可恢复或可重建。
- 恢复后核心业务页面可正常访问。

## 16. 风险与注意事项

| 风险 | 说明 | 建议 |
| --- | --- | --- |
| PHP 版本不兼容 | 禅道版本对 PHP 有兼容范围 | 部署前用测试环境确认 |
| 附件增长过快 | 文档、Bug、反馈图片会占用大量空间 | 使用独立数据盘和对象存储 |
| AI 调用费用不可控 | 文档和图片多时成本上升 | 做限流、缓存、批量队列 |
| 权限泄露 | AI 检索可能跨项目返回结果 | metadata 必须带权限过滤 |
| 向量库膨胀 | 文档多次更新会产生大量 chunk | 更新时删除或失效旧 chunk |
| 模型供应商变更 | 模型 API 可能切换 | 通过模型网关封装调用 |
| 备份不可用 | 只备份不演练等于无备份 | 每月恢复演练 |

## 17. 推荐实施顺序

```text
阶段 1：部署基础禅道
  ↓
阶段 2：配置用户、权限、产品、项目
  ↓
阶段 3：上线 Bug、文档、反馈基础功能
  ↓
阶段 4：部署 AI 服务基础组件
  ↓
阶段 5：接入 Bug 和文档同步
  ↓
阶段 6：接入向量检索和 AI 分析
  ↓
阶段 7：接入反馈 AI 分类和项目风险报告
```

第一阶段上线建议只开放基础禅道和反馈管理；AI 能力先在测试环境跑通同步、向量化和权限过滤，再逐步开放给正式用户。
